STYLES

Styles can be assigned in a number of ways. They can be applied via CSS stylesheets, via <style> blocks or within the `class` attribute.

 

Within each of these areas, except the `class` attribute, the @apply command can be used so that styles are generated based on predefined CSS rules.

Some styles can be generated from the class name. For example  border-b-3 instructs the program to generate a border on the bottom of the element that is 3px in width. More information on this can be found under the Generated Classes section.

Colors, among many others, are another type of style that can be generated. Colors can be applied to many different class rules such as color, background, border, decorations etc. The colors are loaded via a colors.txt file and the user can add any name/color combination they wish. Or the color can be specified via hex code like this text-A355A5.

The class attribute can be overriden in many cases when a template is being used. In this way the template can have a default set of classes and also allow for the individual implementations to override the default class attribute. By default, attributes are merged not overridden, but by setting an attribute option they can be overriden.
e.g. class%o="my-overriding-class".
The "%" character above is used to specifiy attribute options. The o following the % character indicates the option to override the existing template classes with the provided class(s).

Classes defined in the template can be selected by the implementing HTML page; even when they are not global classes. This allows you to define several "local" classes inside the template and then select which one should be applied when using the template.

Local Tag Styles

Style blocks in a template can contain tag names. When the tag style block contains  @local true; the tag style will have a class identifier appended to it. For example given the following style block:

code {
  @local true;
  color: blue;
}

.existing-class {
  border: 2px solid purple
}

Any <code> tag in the template will have the style applied to it. If however the code block already has a class applied to it, then the style class entry above will be turned into a new class. That class will then be inserted into the style sheet and applied to the tag. The insertion is dependent on the order of the class entry in the style block.

See the example below:

code {   @local true;   color: blue; }
.existing-class {   border: 2px solid purple }
.code-sm-ss39549 {   @local true;   color: blue; }
<code class="code-sm-ss39549 existing-class">Content</code>

The order of the classes above matters. If the original code style were below existing-class then when the classes are applied the code block would override any similar rules in existing-class. Generally speaking the tag styles (i.e. code) should come before any local classes to be applied to <code> blocks.
