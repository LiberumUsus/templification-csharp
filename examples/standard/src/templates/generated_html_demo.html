Generating HTML

HTML content can be generated from alternative syntaxes including Markdown and Yorgmayoaml. Element commands are used to indicate that the content within the element is to be interpreted by a processor and then converted into HTML.

Foundations: Simple HTML Generation Syntax

A native method for generating HTML exists to alleviate creating the same type of element over and over with different internal content. The VOID tag can be used to generate content without wrapping the content in a parent element.

For example if a simple list was to be created with string content and the same element with the same attributes over and over the following would do this.

As the requirements become more complicated this feature becomes increasingly pointless. However, for simple lists or repeating elements with different text it is very useful.

<ul class="list-disc">
<void @tag="li"
@separator="|"
@tag-class="text-rose-600">
One|Two|Three|Four|Five
</void>
<ul>
    One|Two|Three|Four|Five

The @tag-<AttrName>attribute can be used to apply any attribute to the generated elements where AttrName is the name of the attribute that should be generated.


It is possible to include tags as part of the list of items to generate from. The wrap-all command must be included in the parent tag. Note that separators are NOT used around tags. The separators in the example are highlighted in green to help show this.

<div @separator="|"
@wrap-all
@tag-class="inline-block text-orange-600 border">
One<b>bold</b>step | in | time.
</div>
Oneboldstep|in|time.
